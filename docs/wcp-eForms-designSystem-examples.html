WCP eForms design system examples

This document references WCP eForms design system specification. It describes the four patient forms in terms of the shared fbXxx component vocabulary. It is derived from design.md rather than source code.

Outpatient outcome

Use the common save/draft/final/RoV shell, but suppress the standard EV sidebar navigation. The form is organised as visual groups around appointment context, consultation outcome, and status/referrals.

Group "Appointment"
  Read-only labels: Organisation, Speciality, Site, Senior clinician, Clinic name, Date and time

Group "Consultation outcome"
  fbGroup Attendance
    attr: required
    fbRadio Attended
      fbGroup Urgent suspected cancer
        attr: required
        fbRadio Yes
        fbRadio No
      fbSCTDiagnosis Working diagnosis
    fbRadio Unable to attend
      fbTextInput Reason
      fbRadio Another appointment already made
        fbPartialDate Date
      fbRadio Send another appointment
      fbRadio No further appointment
        attr: GP/patient letter options
    fbRadio Did not attend
      fbCheck Was not brought
      fbRadio Send another appointment
      fbRadio No further appointment
        attr: GP/patient letter options

Group "Status / referrals"
  Tier 1 mutually exclusive pathway checks: Discharge, See on symptom, Patient initiated follow-up
  Tier 2 pathway checks: Remote monitoring, Tests requested, Add to waiting list, Outpatient treatment planned, Admitted from clinic, MDT review, Follow up appointment
  Non-pathway checks remain enabled: Treatment given in clinic today, Referred to therapies, Referred to another consultant/speciality/hospital

Treatment summary

Treatment summary is generated from a Composer JSON specification and rendered with the same design-system components. Saved values are keyed by component id in form_data, with coded-selector companion values saved as {componentId}_coded.

The Treatment summary implementation should preserve rows, cells, colSpan, required fields, dropdown/radio option labels, text inputs, text areas, MSI selectors, and SNOMED CT diagnosis selectors from the JSON specification.
